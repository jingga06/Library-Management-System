# Library Management System

The Library Management System (LMS) is a web-based application designed to simplify and organize library operations efficiently. This project was developed as the final assignment for the Object-Oriented and Visual Programming (OOVP) course at President University. The system supports book management, borrowing and returning transactions, member administration, and reporting, with separate access roles for admins and members.

## Objectives

- Reduce manual work and improve library management accuracy
- Apply Object-Oriented Programming concepts in a real-world system
- Develop a full-stack Java web application using Servlet and JSP
- Provide role-based access control for Admin and Member users
- Track book availability and transaction history in real time

## Technologies Used

- Java
- Java Servlet & JSP
- MySQL Database
- Apache Tomcat
- NetBeans IDE
- JDBC Connectivity
- MVC-based Client–Server Architecture

## ⭐ Key Features

### Admin (Librarian)
- Secure login and registration
- Manage books and members (CRUD operations)
- Handle borrowing and returning processes
- Filter issued books (returned, overdue, extension requests)
- View complete borrowing and transaction history

### Member (User)
- Member registration and login
- Browse and borrow books online
- Request due date extensions
- Return borrowed books
- View borrowing history and update profile information

## System Highlights

- Role-based access for Admin and Member
- Automatic real-time book availability updates
- Efficient borrowing and returning workflow
- Centralized database for accurate record keeping
- Simple and user-friendly interface

## Quick Setup

1. Install JDK, NetBeans, Tomcat, and MySQL
2. Create the database `library_management_system` and import the SQL file
3. Run the project in NetBeans using Apache Tomcat
4. Access the system via:
```
http://localhost:8080/LibraryManagementSystem
```
